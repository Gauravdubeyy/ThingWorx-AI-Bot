# HUMANIC — Website Documentation

## 1. Overview

**HUMANIC** is a web-based industrial monitoring and reporting platform designed to manage production orders, monitor machine status, track maintenance, audit operational events, and collect MDE (Manufacturing Data Entry) inputs. The application is organized through a side menu that groups features into functional modules such as Order Creation, Machine Info, Maintenance Info, Logs, MDE Screens, and Reports. Each module contains tables, dashboards, and reports supporting daily operational visibility and decision-making. The system presents both real-time and historical information in a structured, navigable format.

---

# 2. Navigation Structure (Side Menu)

## 2.1 Side Menu Modules

The HUMANIC side menu provides quick access to all key functional areas of the application. Each menu item opens a dedicated page that contains either interactive tables, KPI cards, graphs, MDE entry screens, or report views. This modular setup ensures users can jump directly to operational tasks like creating and tracking orders, reviewing machine details, or checking logs. The menu is structured to align with typical manufacturing workflows—from planning and execution to monitoring and reporting.

**Modules Included:**

- Order Creation
- Machine Info

# 3. Order Creation

## 3.1 Purpose

The **Order Creation** module helps users manage current production orders and visualize progress against planning. It includes a detailed table for active orders and a graph comparing produced totals with planned quantities. This section supports production tracking and provides quick visibility into whether output is on schedule. Users can use this view to monitor order completion status and identify deviations early.

### 3.2.1 Description

The **Current Order Table** lists all ongoing or active production orders in a structured grid format. It acts as the primary operational view for tracking what is currently running on the shopfloor. The table is designed to enable quick scanning of order progress, current state, and production-relevant attributes. It supports decision-making for supervisors and planners by consolidating order-level details in one place.

# 4. Machine Info

## 4.1 Purpose

The **Machine Info** module provides a quick overview of all machines through compact information cards. Each card displays key basic machine details at a glance, enabling fast status checking. Users can click a card to open a detailed view for deeper inspection of the selected machine. This section supports quick navigation from fleet-level visibility to machine-level diagnostics and information.

---

## 4.2 Machine Cards (4 Cards)

### 4.2.1 Description

The page contains **four machine information cards**, each representing a machine and showing its basic information. These cards act as interactive shortcuts to machine-specific detail pages. The layout is optimized for quick comparison across machines and helps users identify which machine needs attention. Cards are typically used for summary KPIs, statuses, or static properties depending on the machine configuration.

### 4.2.2 Interaction

- Click on a machine card → opens machine details

# 6. Logs

## 6.1 Purpose

The **Logs** module provides traceability and auditability of operational and system events. It includes telemetry logs for machine/system signals and audit logs for user/system actions and changes. This module is essential for debugging, compliance, investigation, and activity history. Logs are presented as tables to support quick filtering and scanning over time.

---

## 6.2 Telemetry Logs Table

### 6.2.1 Description

The **Telemetry Logs Table** captures telemetry events or signals generated by machines or connected systems. This table helps users review time-based data events, detect abnormalities, and verify system inputs. It supports troubleshooting by providing a historical record of telemetry captured within HUMANIC. Telemetry logs enable transparency into what data was received and when.

---

## 6.3 Audit Logs Table

### 6.3.1 Description

The **Audit Logs Table** records user actions and significant system events for traceability. It helps track changes, identify who performed an action, and verify accountability within the platform. Audit logs are essential for governance and process compliance. This table supports investigations by presenting a chronological history of relevant actions.

# 8. Reports

The **Reports** module provides analytical views for production, downtime, and orders. Each report consolidates operational data into a summarized and structured format for performance tracking. Reports help stakeholders monitor KPIs, validate outcomes, and identify improvement opportunities. These reports are typically used for daily review, weekly analysis, and management summaries.

## 8.2 Production Report

### 8.2.1 Description

The **Production Report** provides summarized information about production output and performance.
